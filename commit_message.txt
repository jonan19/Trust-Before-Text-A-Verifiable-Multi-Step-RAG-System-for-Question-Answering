Refactor Qdrant hybrid retrieval, strengthen startup checks, and update deps

Overview:
- Integrate and refine Qdrant hybrid retrieval (dense + sparse prefetch + ColBERT multivector rerank).
- Add model-consistency checks and automatic re-ingest on startup for both ChromaDB and Qdrant.
- Update dependencies required for ColBERT, SentenceTransformers and Qdrant client.

Files changed:
- qdrant_retrieval.py: implements ColBERT multivector encoding, sparse encoder fit/save, dense/sparse/multi retrieval paths, RRF reranking, caching, manifest handling, and robust error messages when collection is missing.
- main.py: adds model consistency checks, auto-ingest trigger when manifest model mismatches, improved status/comparison modes and clearer startup output.
- requirements.txt: add/update packages (qdrant-client, sentence-transformers, transformers, safetensors, huggingface-hub, etc.) required by the new retrieval components.

Migration / Notes:
- After pulling, rebuild the Qdrant collection if using the Qdrant retriever:
  python main.py --ingest --retriever qdrant

Testing:
- Manual local runs and status checks recommended. No breaking API changes expected for existing ChromaDB-only workflows.
